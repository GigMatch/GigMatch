GigMatch Timeline

  • Day 1: Project planning

    7 July, 2020

    On day one, we as a team came together and decided what we wanted GigMatch to be. We created user stories and delved into the minds of a GigMatch user and what they would want to see or do in a "gigmatching" website. Moving forward we created a feature list containing all needed features for the site, as well as a roadmap for future additions.

    To fully understand GigMatch, we developed a series of wireframes and storyboards to visualize what our application would come to be. We created the structure and outline that helped portray the flow of a user from setting up their account to creating that last post. This was all done to help contribute to a fully functional GigMatch.

  • Day 3: Database design and creation

    9 July, 2020

    On day 3, we the GigMatch developers designed and setup the database foundation for which our site was to be built upon. We referred to our previous wireframe and feature list mock-ups to blueprint a database that would work cohesively with the user stories we created. Different tables were setup in combination with the Spring framework to map out what information was to be stored and from what features.

  • Day 4: Development starts

    10 July, 2020

    On day 4, development was on its way. Database design was finished and feature creation had begun. The GigMatch developers began to map out and shape the way the features would work and on what views according to the storyboards. Created with Java, following the MVC pattern and utilizing the Springboot framework the developers began coding.

  • Day 11: GigMatch is navigable

    17 July, 2020

    On day 11, the GigMatch application was completely navigable with users being able to signup or login and start using GigMatch. The developers reached this milestone and once again referred back to their user stories to start using the application they built. Here they recorded and documented any errors or issues with both code and design for the application.

  • Day 20: GigMatch is fully operational

    27 July, 2020

    On day 20, GigMatch was deployed and fully operational. After intensive review, the application was approved for new users to start finding their gigs. The team revised and adjusted based on previous user feedback to bring their best version to date.

    Going forward, the software developers of GigMatch have many plans to both improve the design and backend development for the application. Features such as an interactive calendar, notifications, direct messaging system, and the ability to follow other users are all to be expected in GigMatch 2.0
